 Trabalho desenvolvido por João Rosário e Diogo Nunes.
 O nosso programa realiza as funções todas pedidas, claro que estão adaptadas as classes que usamos mas pensamos ter atingido um bom resultado.
 O trabalho começa por pedir o nome do ficheiro da agencia, no caso é finalx (o programa sozinho acrescenta o .txt portanto deve ser introduzido sem este).
 Ao longo do programa o 0 serve sempre de botão para voltar exceto num caso especifico que é quando queremos ver os pacotes dentro do menu de venda. Visto que nenhum pacote tem o 
id igual a 0 este serviu bem para o proposito.
 Ao iniciar então a agencia temos um primeiro menu com opções mais gerais no caso o menu de pacotes, clientes, informações da agencia, Vendas.
 No primeiro temos todas as operações sobre pacotes, nomeadamente: criar, apagar, modificar e ver os pacotes. Dentro de modificar podemos alterar qualquer atributo deste.
Dentro de ver pacotes podemos filtrar por disponibilidade, e por data e tambem possivel pesquisar um pacote por ID.
 No menu clientes podemos criar, apagar e ver clientes. Dentro deste ultimo (ver clientes) é possivel ver a ficha individual de cada um e dentro da mesma alterar qualquer 
atributo do cliente, ver as informações gerais e ainda obter uma sugestão e comprar um pacote com um destino incluido nos N locais mais visitados.
 Nas informações da agencia é possivel ver as informaçoes gerais(dinheiro acumulado, nif, nome dos ficheiros que contem a informação,etc) e tambem é possivel ver os 
N locais mais visitados.
 No ultimo Menu temos as vendas em que nos é possível vender um pacote a um cliente.
 O funcionamento das funções mais complexas esta no programa como comentário.
